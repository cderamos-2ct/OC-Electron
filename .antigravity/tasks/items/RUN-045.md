## Durable Intake Buckets / Workflow
- `urgent`
  - Same-day decision or blocking issue.
  - Gmail signal: `🔴 Urgent`, high-consequence finance/order/support traffic, or human messages that will stall work if ignored.
  - Messages signal: inbound human thread that is actively blocking a decision or next move.
- `needs reply`
  - Latest message is inbound from a human and there is no reply / owner capture yet.
  - If the answer is trivial, prepare a draft.
  - If the answer needs lookup/work, promote to a `RUN-*` task the same day.
- `follow-up`
  - Christian already replied or promised an action, but the thread still needs to come back.
  - This is the missing Gmail bucket in practice right now: `📌 Follow-Up` exists but had `0` hits in the first audit.
  - Use either the Gmail label or a canonical task if the follow-up requires real work.
- `waiting`
  - Third party owes the next move; keep it out of `unread` and track the owner/date in task state or daily brief.
  - Example from the first audit: lease-review thread, AR-overdue thread waiting on Andy, OneVision waiting for either artifacts or support response.
- `reference`
  - Meeting recaps, receipts, payment confirmations, invites, tracking notices, and similar non-action traffic.
  - Keep accessible, but do not let it live as attention debt in inboxes.
- `junk`
  - Promotions, scams, cold outreach, and irrelevant automation.
  - No tasking; archive/delete during future hygiene passes.

## First Operating Rule Set
1. Sweep `urgent` + `needs reply` in Gmail and Messages first.
2. Any non-trivial ask becomes either a draft-prep item or a canonical `RUN-*` task immediately.
3. After any reply/promise, move the item into `follow-up` or `waiting` instead of leaving it as inbox residue.
4. Keep `reference` visible but out of attention-critical queues.
5. Never send externally without approval; drafts, owner routing, and task promotion can happen proactively.
