# FOLLOW_UP_QUEUE.md

## Purpose
Track follow-ups that emerge from meetings, emails, task reviews, and executive workflows.

This queue is for things that need a response, check-in, clarification, or next-step action — not for every generic to-do.

---

## Status definitions
- **captured** — follow-up identified, not yet shaped
- **drafted** — follow-up text/approach exists
- **ready** — ready to send/execute/review
- **sent** — follow-up completed/sent
- **waiting** — waiting on someone else
- **closed** — no further action needed
