# Voice Layer Roadmap

## Goal
Move toward a low-friction voice interface where Christian can interact with CD through AirPods and spoken conversation.

## Desired outcome
- always-available enough for practical use
- can brief, capture, remind, and answer contextually
- integrates with dashboard/task/calendar/meeting systems

## Dependencies
- working persistent digest
- reliable alerts/attention model
- meeting/task/email/calendar data integration
- speech-to-text
- text-to-speech
- interruption logic

## Priority
Voice should follow the assistant data/notification layer, not precede it.
