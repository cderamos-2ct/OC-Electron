# Executive Operating Model

## Purpose
Define how CD should support Christian across:
- tasks
- reminders
- calendar
- meeting prep
- meeting notes
- follow-ups

The goal is not to create more systems.
The goal is to make the existing systems work together while giving Christian one operator layer that keeps the madness organized.

---

## 1) System roles

### Calendar = commitments
Use the calendar as the source of truth for:
- meetings
- appointments
- time-bound commitments
- scheduled events

### Reminders / task system = actions
Use Reminders / task systems for:
- follow-ups
- prep tasks
- calls
- admin to-dos
- things that do not belong as calendar events

### Dashboard = operator layer
Use the dashboard to show:
- what is next
- what needs prep
- what is overdue
- what came out of meetings
- what follow-ups are waiting
- what deserves attention now

The dashboard should summarize and prioritize.
It does not need to be the original source of truth for every item.

---

## 2) Task model

### Recommended split

#### Tactical/personal tasks
Best home:
- Apple Reminders (good mobile/native capture)

Examples:
- call Eric Rosenfeld
- review Drive changes
- finish workbook
- groceries / household reminders

#### Strategic/project tasks
Best home:
- dashboard + project docs/backlogs
- optionally mirrored into a task system if helpful

Examples:
- CTO learning artifacts
- dashboard module work
- experiment backlog refinement
- cross-entity planning work

### Rule
Do not force every strategic workstream into a flat personal reminder list.
That makes the task system dumb and noisy.

---

## 3) Daily calendar support

### Daily briefing goal
Every day, Christian should be able to get a next-day or same-day executive brief that shows:
- meetings/events
- time + participants if available
- prep needed
- relevant docs/notes
- likely follow-ups

### Suggested cadence

#### Night-before brief
- tomorrow's calendar summary
- what needs prep tonight
- what can wait

#### Morning brief
- today's meetings
- top prep items
- anything unusual / missing / risky

#### Pre-meeting prep alerts
For important meetings:
- 60 minutes before → prep reminder
- 15 minutes before → final reminder / links / materials

---

## 4) Meeting workflow

### Before meeting
CD should help surface:
- what the meeting is
- who is involved
- what context/docs matter
- what Christian should be ready to ask / decide

### During meeting
Preferred path:
- use transcript/audio capture from the meeting platform or a local recording workflow

Potential inputs:
- Zoom transcript
- Google Meet transcript
- Teams transcript
- local audio recording + transcription

### After meeting
CD should convert the transcript/notes into:
- concise meeting summary
- decisions made
- action items
- owners
- deadlines/follow-ups
- partner/team-shareable notes if needed

---

## 5) Follow-up model

Every meeting should eventually produce one of these:
- no action
- a reminder/task
- a follow-up draft
- a strategic note
- an experiment / project task

### Follow-up outputs
- personal task for Christian
- shareable recap for team/partners
- questions needing response
- dashboard update

---

## 6) Notification model

### What should trigger proactive notifications
- important calendar event tomorrow
- important prep item before a meeting
- a meeting produced action items that are still uncaptured
- an urgent email arrived
- a real blocker exists on active work

### What should not trigger constant interruption
- every random message
- low-value promo emails
- duplicate status updates
- busywork with no decision/action attached

---

## 7) Best implementation path

### Phase 1 — briefing and visibility
- daily calendar brief
- task/reminder visibility
- important email surfacing
- prep reminders

### Phase 2 — meeting capture
- transcript/audio ingestion
- note/action extraction
- follow-up creation

### Phase 3 — dashboard integration
- unified operator view across:
  - calendar
  - reminders/tasks
  - meeting outputs
  - follow-up queue

---

## 8) Rule
The operator layer should reduce fragmentation.

It should not make Christian wonder:
- where a task lives
- what meeting needs prep
- what came out of a meeting
- what needs follow-up

That is CD's job.
